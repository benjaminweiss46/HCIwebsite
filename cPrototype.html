User Manual

Design Evolution

Slow Chat

The slow chat button was a feature that allowed to slow down the chat recieved by zoom. This is done so the host never misses a comment at the expense of a potential delay on the hosts chat. While this option was good in practice if confused users in testing who didn't understand what this button did. Even with some explination some users didn't get what it really did. So this was completely removed.

Tool Tips

While we may have believed our interface was so easy to use it needed no explanation. This proved to be incorrect, some user had no idea what to do. To combat this we have included tooltips on some of the buttons to indicate what actions the button performs.

More Options

While we wanted have more than 4 graph options, and give users maximal optionality we didn't give them a way to get more in practice. Therefore we added a button to the graph options so that users can select from a much larger range of options.

Graph Customization

We initially expected users to customize the graphs by clicking on graph elements. Not many users understood this interation. Therefore we moved graph customization to its own section, with labels and input boxes so users know exactly how to customize the graph.
